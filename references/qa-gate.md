# QA Gate(强制后置门禁)

渲染出 PNG 后**必须**跑这个门禁再交付。

**关于"看图"的诚实约定:** 你能机器核验几何与像素(四角白、墨量、尺寸、比例),但通常**无法用工具看懂生成场景的语义**——没有 vision/`vision_analyze` 工具,`browser_navigate` 对裸 PNG 只回空快照(它只读 DOM)。所以**不要谎称"我看过图了没问题"**。正确做法:① 跑下面机器检查;② 把所有分节图拼成 contact sheet 发用户;③ 明说机器指标已验、语义判断(小黑是否真在做动作 / 有没有混进文字)交给用户,并主动提出可单独回炉某一张不动其他张。**用户就是那只眼**,把流程设计成依赖用户核验,别假装自己核验了。

PIL 检查用**系统 `python3`**(自带 Pillow)。`execute_code` 沙箱**没有 PIL**(`from PIL import Image` 会 ModuleNotFoundError),改用 terminal 调 `python3`,或 `uv run --with pillow python3`。

## 0. 机器可查(先跑)

```bash
python3 -c "from PIL import Image;w,h=Image.open('out.png').size;print(f'{w}x{h} 1:{h/w:.2f}')"
```

- [ ] 宽 ≈ 3240px(1080 @ 3x)。
- [ ] 纵横比落在 **1:1.4 – 1:1.9**。> 1.9 → 精简一节 / 收紧间距 / 拆成第二张图。**不许靠缩窄宽度凑比例。**

## 1. 必过项(交给用户核验 / 你机器能查的尽量查)

- [ ] 背景是**干净纯白**,且小黑场景与页面之间**无可见接缝、无边框、无阴影、无色块**。
- [ ] 版面是**无框并排流**,不是带边框的卡片网格 / 仪表盘 / PPT。
- [ ] 每节都有小黑,且小黑在**做核心动作**——去掉它该节就不成立。
- [ ] 隐喻是**为这篇文章新发明的**,没有复刻 ian examples 旧构图。
- [ ] 小黑插图与文字**左右并排、奇偶交替换边**,有留白节奏。
- [ ] 文字**全部清晰**(因为文字在 HTML);**生成的小黑图里没有任何文字/字母/数字**。
- [ ] 批注少而短(每节 1–2 处、2–8 字),颜色克制:红=问题/结果,橙=主路径,蓝=补充。
- [ ] 手机尺寸下主文字可读(标题/要点字号没被压到糊)。

## 2. 失败信号(出现就回炉)

- 小黑像吉祥物 / 表情包 / 可爱卡通,或只站在角落当装饰。
- 画面像 PPT、课程课件、正式流程图;出现成排边框卡片或密集箭头网格。
- 生成图里冒出文字、标签、标题(尤其左上角"流程图 / 系统架构图 / 常见坑"这类)。
- 背景有米色 / 纸纹 / 渐变 / 阴影 / 噪点,或小黑图块周围有可见方框/接缝。
- 元素太多、太满,主体超过画面 ~60%。
- 中文要点变成大段解释。
- 纵横比 > 1.9(手机上被压成细长条会糊)。
- 整张和 ian `assets/examples/` 旧案例构图过于相似。

## 3. 回炉方法

- **太普通** → 让小黑成为动作主体,换一个更奇怪但成立的隐喻(重生成该节)。
- **太 PPT / 有边框感** → 去掉 `.shot` 周围任何 border/shadow;确认 section 无边框;靠留白分节。
- **太可爱** → 提示词强调 deadpan、blank serious、not cute、not mascot,重生成。
- **图里有字** → 重生成并再次强调 NO text;或合成前把字抹掉补白。
- **太高** → 删/并一节,或收紧 `.wrap` gap 与 `.shot` 比例;实在不行拆成两张图。
- **和旧案例像** → 保留核心意思,换掉主物件和小黑动作。

## 交付判断

高质量图应让读者先觉得"有点怪",然后 1 秒内看懂结构。第一眼像教程页 / PPT,而不是白纸上的怪诞产品草图,就不合格。

Telegram 交付记得发**文件**(`scripts/send_telegram.sh`),不要走 sendPhoto 被压糊。
